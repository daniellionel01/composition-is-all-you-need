# Reference library

Use these references as sources of principles, not costumes to copy. The goal is to borrow discipline, not imitate a brand without context.

## Stripe

### What to learn

- Makes complex technical products feel approachable.
- Uses refined gradients and dimensionality with restraint.
- Combines dense information with generous structure.
- Strong typographic hierarchy and precise spacing.
- Product illustrations and diagrams clarify abstract systems.
- Documentation and marketing share a coherent visual language.

### Borrowable principles

- Use color and depth to explain layers, flows, or systems.
- Pair technical credibility with warmth.
- Make dense content feel navigable through strong sectioning.
- Use gradients as atmosphere or hierarchy, not as decoration everywhere.
- Keep text crisp and direct.

### Use when

- designing SaaS/product pages
- explaining workflows or systems
- creating technical reports for nontechnical stakeholders
- making dashboards feel premium without losing clarity

### Avoid

- generic purple-blue gradient hero sections
- fake 3D blobs with no meaning
- overcomplicated decorative backgrounds

---

## Linear

### What to learn

- Calm, precise, and confident.
- Excellent dark-mode discipline.
- Fine borders, subtle contrast, and restrained depth.
- Clear product hierarchy without heavy decoration.
- Dense interfaces that still feel elegant.

### Borrowable principles

- Use near-black palettes with carefully tuned contrast.
- Let typography and spacing do more than color.
- Use thin rules and quiet surfaces for structure.
- Make interaction states crisp and minimal.
- Favor high signal, low ornament.

### Use when

- designing technical dashboards
- creating engineering or operations tools
- building dark-mode artifacts
- presenting roadmap, issue, product, or performance data

### Avoid

- making everything dark just to seem premium
- contrast too low for readability
- copying the exact Linear look without reason

---

## Apple

### What to learn

- Ruthless focus.
- One hero idea at a time.
- Cinematic spacing and pacing.
- Material restraint.
- Product details presented as inevitable and obvious.
- High confidence through subtraction.

### Borrowable principles

- Give important content room.
- Use fewer words and stronger hierarchy.
- Create drama through scale and whitespace.
- Avoid clutter around the main idea.
- Treat transitions between sections as pacing, not filler.

### Use when

- creating product presentations
- designing high-level executive decks
- making premium landing pages
- producing launch or announcement artifacts

### Avoid

- overusing huge empty space for low-importance content
- vague inspirational copy
- making dense analytical content too sparse to be useful

---

## Bauhaus

### What to learn

- Form follows function.
- Geometry can create clarity and energy.
- Primary shapes and strong alignment can feel timeless.
- Asymmetry can be structured rather than chaotic.
- Decoration should emerge from construction.

### Borrowable principles

- Use simple geometric systems.
- Create rhythm with circles, bars, blocks, and grids.
- Use primary or near-primary accents sparingly.
- Make structure visible.
- Let layout express the function of the content.

### Use when

- designing posters, section dividers, infographics, or educational visuals
- giving a deck a strong graphic language
- creating diagrams with geometric clarity

### Avoid

- turning everything into primary-color cosplay
- using shapes that do not clarify the content
- sacrificing readability for composition

---

## Swiss design / International Typographic Style

### What to learn

- Grid discipline.
- Objective clarity.
- Typography as the main visual tool.
- Asymmetric layouts with strong alignment.
- Minimal ornament.
- Clear hierarchy through scale, weight, and position.

### Borrowable principles

- Build on a grid.
- Use fewer type sizes but make the differences meaningful.
- Align everything deliberately.
- Use ragged-right text for readability.
- Make whitespace active.
- Let rules, labels, and modules create order.

### Use when

- creating reports, dashboards, data-heavy pages, and professional documents
- improving credibility
- making dense information easier to navigate

### Avoid

- sterile layouts with no point of view
- tiny type in the name of minimalism
- grids that make every section feel identical

---

## Editorial design

### What to learn

- Strong pacing between headline, deck, body, figure, caption, and pull quote.
- Content-specific layouts.
- Margins and whitespace create authority.
- Captions and sidebars can carry important context.
- A single spread or page can have a memorable composition without harming readability.

### Borrowable principles

- Use title, subtitle, byline/source, and date as a designed opening block.
- Add pull quotes or key findings to break long text.
- Use captions that interpret visuals.
- Use margin notes for context and caveats.
- Create rhythm across sections.

### Use when

- designing PDFs, reports, long-form HTML, whitepapers, and strategic memos

### Avoid

- magazine drama for routine internal documents
- too many pull quotes
- text columns that are too narrow or too wide

---

## Financial and institutional reporting

### What to learn

- Trust comes from consistency, restraint, and precision.
- Tables can be beautifully designed without looking decorative.
- Footnotes, units, and assumptions matter.
- Conservative design can still be polished.

### Borrowable principles

- Use stable grids.
- Keep colors quiet and semantic.
- Format numbers consistently.
- Make totals and subtotals clear.
- Use thin rules and whitespace rather than heavy boxes.
- Place source notes consistently.

### Use when

- designing board reports, financial dashboards, operating reviews, spreadsheets, investor materials

### Avoid

- playful colors for serious financial risk
- hiding caveats
- over-styled charts that reduce trust

---

## Luxury and premium visual language

### What to learn

- Premium often means fewer elements, better materials, and more confident spacing.
- Typography, photography, and negative space carry the tone.
- Color palettes are usually restrained.

### Borrowable principles

- Use generous margins.
- Use high-quality typography.
- Use one focal point.
- Prefer texture, tone, and composition over decoration.
- Make interactions feel quiet and deliberate.

### Use when

- designing high-end brand pages, executive summaries, investor decks, premium product presentations

### Avoid

- low contrast that hurts readability
- empty luxury styling on utilitarian tools
- fake prestige through gold accents and thin serif type alone

---

## Data journalism

### What to learn

- Charts are explanatory, not just illustrative.
- Annotation can guide attention.
- Scrollytelling and sequencing help complex ideas.
- The best chart is often simple but well-captioned.

### Borrowable principles

- Lead with the question.
- Annotate the key insight directly.
- Use small multiples for comparison.
- Add context through benchmarks and reference lines.
- Keep chart furniture quiet.

### Use when

- designing reports, public-facing dashboards, analytical explainers, policy or research summaries

### Avoid

- chart variety for its own sake
- decorative complexity
- unsupported editorial claims

---

## Government and civic design

### What to learn

- Accessibility and clarity are non-negotiable.
- Plain language builds trust.
- Design systems can make large amounts of information navigable.

### Borrowable principles

- Use clear labels.
- Use high contrast.
- Make forms and tables obvious.
- Avoid jargon.
- Prioritize function over novelty.

### Use when

- designing public information pages, compliance reports, forms, operational dashboards, nonprofit material

### Avoid

- overly trendy visuals
- low contrast
- ambiguous interaction states

---

## Choosing a reference direction

Pick references based on the artifact's job:

- **Technical product dashboard:** Linear + Swiss + financial reporting
- **Sales dashboard:** Stripe + financial reporting + data journalism
- **Executive deck:** Apple + Swiss + editorial
- **Investor deck:** Apple + financial reporting + luxury restraint
- **Operational spreadsheet:** financial reporting + civic clarity
- **Research PDF:** editorial + Swiss + data journalism
- **Marketing HTML page:** Stripe + editorial + premium language
- **Public report:** civic design + data journalism + Swiss
- **Creative presentation:** Bauhaus + editorial + Apple pacing

## Reference translation examples

### Generic request: "Create a sales dashboard"

Reference translation:

- Use financial reporting discipline for trust.
- Use data journalism annotations to point out pipeline risk.
- Use Stripe-like warmth to avoid sterile enterprise gray.
- Avoid equal card grids; create a hero revenue/pipeline area plus exception detail.

### Generic request: "Make a presentation deck"

Reference translation:

- Use Apple pacing for one idea per slide.
- Use Swiss grids for alignment.
- Use editorial titles that make claims.
- Use Bauhaus geometry only if the subject benefits from a strong graphic system.

### Generic request: "Create an HTML report"

Reference translation:

- Use editorial design for reading flow.
- Use data journalism for charts and annotations.
- Use Stripe-like sectioning for complex systems.
- Use restrained color and careful captions for credibility.
